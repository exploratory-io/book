# Exploratory ノート自動作成プロジェクト

## プロジェクト概要

Exploratoryの機能に関するノート記事を、サブエージェントを使って自動作成する仕組みです。

## ディレクトリ構造

```
Create_Notes/
├── CLAUDE.md              # このファイル（プロジェクト設定）
├── agents/                # 各エージェントの指示書
│   ├── official_researcher.md   # 公式リサーチャー
│   ├── general_researcher.md    # 一般リサーチャー
│   ├── writer.md                # ライター
│   └── editor.md                # 編集者・レビュアー
├── output/                # 作成された記事の出力先
└── .claude/
    └── skills/
        └── create-note/   # オーケストレーションスキル
            └── index.md
```

## 基本ルール

- 必ず日本語で応対する
- ノート作成時は `/create-note` スキルに従う
- Exploratoryのノートページの取得には `exploratory-web-fetch` グローバルスキルを使う
- 出力先は `./output/` フォルダ

## 記事作成の流れ

1. ユーザーがタイトルとコンテキストを提供する
2. 公式リサーチと一般リサーチを**並列**でサブエージェント実行する
3. リサーチ結果を統合してライターサブエージェントが執筆する
4. エディターサブエージェントがレビューする
5. 合格すれば `output/` に保存、差し戻しなら最大2回再執筆する

## Exploratoryページの取得ルール

Exploratoryのノートページ（`exploratory.io/note/`）は、URLの末尾に `/note_content/note.html` を付けてWebFetchする。詳細は `exploratory-web-fetch` スキルを参照。

## 記事のまとめセクション

- 必ず**文章形式**（散文）で書く。箇条書きは使わない
- 本文の内容と整合性を保つ
- 2段落構成: 第1段落は機能概要、第2段落は何が嬉しいのか

## 英語版作成

- 対応する英語版の元記事がある場合、そのスクリーンショット画像パスを使用する
- 変数名やメニュー名は英語版UIの表記に合わせる

## ユーザー修正の扱い

- ユーザーが修正版を提供した場合は、常にその修正版をベースとして尊重する
- 修正の意図を理解し、それに沿った編集を行う
