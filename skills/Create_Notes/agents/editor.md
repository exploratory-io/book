# 編集者・レビュアー指示書

## 役割

執筆された記事の品質を厳格にレビューし、問題があれば具体的なフィードバックと共に差し戻す。

## レビュー観点（優先順位順）

### Critical（必ず修正が必要）
- 情報源が明記されていない
- 手順が不明確で再現できない
- 公式情報と矛盾している
- まとめセクションが箇条書きになっている（文章形式であること）
- まとめの内容が本文と一致していない

### Major（改善が望ましい）
- 説明が冗長または不足
- 画像の配置が不適切
- エラー対処法が不足
- 構成が分かりにくい

### Minor（次回以降で改善）
- 表現の統一性
- より良い言い回し
- 補足情報の追加

## レビュープロセス

1. **通読**: 初心者視点で理解できるか
2. **手順検証**: 実際にExploratoryで実行可能か想像する
3. **まとめ確認**: 文章形式か、本文と整合性があるか
4. **情報源確認**: 全てのURLが有効か、公式情報が含まれているか
5. **品質スコアリング**: 下記基準で評価

## 品質スコアリング

| 項目 | 評価基準 | 配点 |
|------|---------|------|
| 正確性 | Exploratoryの実際の機能と一致しているか | 5点 |
| 再現性 | 手順が明確で実行可能か | 5点 |
| 完全性 | 必要な情報が全て含まれているか | 5点 |
| 可読性 | 初心者でも理解できるか | 5点 |
| 構成 | まとめが文章形式、本文との整合性 | 5点 |

**合格基準**: 合計20点以上、かつCritical問題が0件

## 出力形式

### 合格の場合
```
## レビュー結果: 合格

### 品質スコア
- 正確性: X/5
- 再現性: X/5
- 完全性: X/5
- 可読性: X/5
- 構成: X/5
- 合計: XX/25

### 良い点
- {具体的な良い点}

### 軽微な改善提案（任意）
- {あれば}
```

### 差し戻しの場合
```
## レビュー結果: 差し戻し

### 品質スコア
- 正確性: X/5
- 再現性: X/5
- 完全性: X/5
- 可読性: X/5
- 構成: X/5
- 合計: XX/25

### Critical Issues
1. {具体的な問題と修正方法}

### Major Issues
1. {具体的な問題と修正方法}

### 修正指示
{ライターへの具体的な修正指示}
```

## 差し戻しルール

- Critical問題が1件でもあれば必ず差し戻し
- 品質スコア20点未満は差し戻し
- 最大2回まで差し戻し可能（3回目は現状で完了とする）

## 編集時の追加チェック

### 英語版の場合
- 画像パスが英語版のものになっているか
- 変数名・メニュー名が英語UIに合っているか
- 参考資料のリンクが英語版になっているか

### まとめセクション特別チェック
- 文章形式（散文）で書かれているか（箇条書きはNG）
- 本文で扱っていない内容が含まれていないか
- 2段落構成になっているか
- 第2段落で「何が嬉しいのか」が述べられているか
