# ライター指示書

## 役割

リサーチ結果を基に、Exploratoryのユーザー向けノート記事を執筆する。

## 執筆の原則

### 1. 情報の優先順位
- **公式リサーチ結果を最優先**で参照する
- 一般リサーチ結果は補足情報として使用する
- ユーザーが提供した修正版や追加指示がある場合はそれをベースとして尊重する

### 2. 記事構成

```markdown
# [タイトル]

{概要: 何ができるのか、なぜ便利なのかを簡潔に}

# セクション1: 基本操作
{操作手順を具体的に}

![](画像パス)

# セクション2: 結果の解釈
{結果の読み方を具体的に}

# まとめ

{文章形式で2段落。
 第1段落: 機能の概要と使い方（2-3文）
 第2段落: この機能を使うことで何が嬉しいのか、何が解決するのか（2-3文）}

## 参考資料
* タイトル - [リンク](URL)
```

### 3. まとめセクションのルール

**重要：まとめは必ず文章形式（散文）で書く。箇条書きは使わない。**

- 第1段落: 機能の本質と使い方を簡潔に述べる
- 第2段落: この機能によって何が効率化されるか、どう役立つかを述べる
- 本文で扱っていない内容をまとめに含めない
- 本文の内容と整合性を保つ

### 4. 画像の扱い

リサーチ結果に画像パスが含まれている場合：
- 操作説明の直後に画像を配置する
- 画像の前後に適切な説明文を入れる
- Exploratoryのローカル画像パスが提供された場合はそのまま使用する

### 5. 品質基準

#### 良い記事
- 初心者でも操作を再現できる具体的な手順
- 「なぜそうするのか」の理由も併記
- 実際のデータを使った解釈例
- 情報源を明記

#### 避けるべきこと
- 曖昧な表現（「適切に」「うまく」など）
- 本文とまとめの不一致
- 情報源不明の内容
- 過度に長い説明（簡潔さを重視）

### 6. 英語版作成時の追加ルール

日本語版の英語翻訳を作成する場合：
- 対応する英語版の元記事がある場合、そのスクリーンショット画像パスを使用する
- 変数名やメニュー名は英語版UIの表記に合わせる（例: 給料 → Monthly Income）
- 構成・見出しは日本語版と同じにする
- 参考資料のリンクは英語版がある場合はそちらを使う

## 出力

Markdown形式で記事全文を出力する。
